    Beam me down, Santa! Today we spelunked a tachyon manifold with splitters and smoke. We tackled two related simulations: a classical beam-splitting grid (Part 1) and then its quantum many-worlds cousin (Part 2).

    Below I explain both parts, the algorithms, the key insights, and why this is efficient.

    Problem summary

    The input is a rectangular grid with characters:

    • S: the entry point of a downward-moving tachyon beam
    • .: empty space, beam passes through straight down
    • ^: splitter; a downward beam is stopped here and two new beams emerge from the immediate left and right positions in the next row (if in-bounds)

    Beams always move downward one row at a time.

    • Part 1: Multiple classical beams travel simultaneously. When a beam hits a splitter, it stops and spawns two beams to left/right one row below. Multiple beams that land on the same cell just overlap and continue as one beam. Compute the total number of times a beam is split (i.e., the total number of splitters encountered by at least one active beam).

    • Part 2: A single quantum particle travels, but time splits at each splitter so it takes both left and right. Timelines that later land at the same spot do not cancel out; they remain distinct timelines (different histories) even if they share the same future path. Count the total number of resulting timelines after the particle completes all possible journeys (leaving the grid or going out-of-bounds).

    Part 1 â€” Classical beams with merging

    Key model:

    • Process the grid row by row starting below S.
    • Maintain the set of active columns in the current row where a beam exists.
    • Transition to the next row:
    • If the cell below an active column is '.', the beam continues straight.
    • If it's '^', count one split and emit two beams to adjacent columns (left/right) of the next row (in-bounds only). The original beam stops.
    • Use a set for the next row to automatically merge overlapping beams.

    This exactly mirrors the provided example frames and efficiently simulates all beams while merging.

    Why it works:

    • Each row depends only on the previous rowâ€™s active columns; no backtracking.
    • Merging is naturally handled by the set structure.
    • Time complexity O(R*C) in practice, since the set size is at most C and we process R rows.

    Gotcha avoided:

    • The splitting count is the number of split events (i.e., number of times an active beam meets a '^'), not the number of new beams created.

    Part 2 â€” Quantum timelines (many-worlds)

    Interpretation nuance:

    • A quantum particle splits into two timelines at each '^'. Those different histories remain distinct even if they later occupy the same grid cell. So we must count timelines with multiplicity, not deduplicate after merges.

    This maps to a DAG path-counting problem:

    • Nodes are positions (r, c).
    • From (r, c) you transition to the next row: either straight down if the next cell is '.', or both diagonals if itâ€™s '^'.
    • Multiple ways to reach the same node produce multiple distinct histories (add counts).
    • Finishing conditions:
    • Stepping straight down from the last row exits (counts toward answer).
    • Splitting at the edge can send a branch out-of-bounds immediately; those contribute to the total too.

    Dynamic Programming:

    • Keep a 1D array curr[c] = number of ways to be at (r, c) at current row.
    • Build nxt for row r+1 by distributing counts according to the character in the next row.
    • Accumulate exits when branches go out-of-bounds or walk off the bottom.

    This is again O(R*C) and uses Python big integers for arbitrary-precision counts.

    # Advent of Code 2025 Day 7 - Laboratories - Part 1
# Read input grid and simulate downward beams with splitters '^'.
# Count how many times a beam is split (i.e., how many '^' cells are reached by at least one beam).

def solve(data: str) -> int:
    grid = [list(line.rstrip('\n')) for line in data.splitlines() if line.strip('\n') != '']
    if not grid:
        return 0
    R, C = len(grid), len(grid[0])
    # Find S position
    rs = cs = None
    for r in range(R):
        for c in range(C):
            if grid[r][c] == 'S':
                rs, cs = r, c
                break
        if rs is not None:
            break
    if rs is None:
        # No start found; nothing happens
        return 0
    active = {cs}
    splits = 0
    # Process rows below S
    for r in range(rs + 1, R):
        next_active = set()
        # Using set to merge overlapping beams
        for c in active:
            ch = grid[r][c]
            if ch == '^':
                splits += 1
                if c - 1 >= 0:
                    next_active.add(c - 1)
                if c + 1 < C:
                    next_active.add(c + 1)
                # Beam at c stops (do not carry c over)
            else:
                # empty space or other char ('.' or maybe 'S' though unlikely): continue straight
                next_active.add(c)
        active = next_active
        if not active:
            # All beams have exited or been stopped
            break
    return splits

if __name__ == '__main__':
    import sys
    with open('input.txt', 'r', encoding='utf-8') as f:
        data = f.read()
    ans = solve(data)
    print(ans)

def solve(data: str) -> int:
    grid = [list(line.rstrip('\n')) for line in data.splitlines() if line.strip('\n') != '']
    if not grid:
        return 0
    R, C = len(grid), len(grid[0])
    rs = cs = None
    for r in range(R):
        for c in range(C):
            if grid[r][c] == 'S':
                rs, cs = r, c
                break
        if rs is not None:
            break
    if rs is None:
        return 0
    # DP counts per row to save memory
    curr = [0] * C
    curr[cs] = 1
    exits = 0
    for r in range(rs, R-1):
        nxt = [0] * C
        for c, ways in enumerate(curr):
            if ways == 0:
                continue
            ch = grid[r+1][c]
            if ch == '^':
                # split
                if c-1 >= 0:
                    nxt[c-1] += ways
                else:
                    exits += ways
                if c+1 < C:
                    nxt[c+1] += ways
                else:
                    exits += ways
            else:
                # straight down
                nxt[c] += ways
        curr = nxt
    # At last row, moving one more step exits for any remaining ways
    exits += sum(curr)
    return exits
