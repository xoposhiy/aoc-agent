    The cephalopods have led us to a research wing, and naturally, we stepped into a teleporter. Now we are fixing a Quantum Tachyon Manifold! This puzzle had us tracing beams through a grid of splitters, evolving from a simple simulation to a multiverse path-counting problem.

    Part 1: The Leaking Manifold ðŸ’¥

    We are given a grid with a start point S, empty space ., and splitters ^.

    • Physics: Beams fall downward.
    • Splitters: When a beam hits ^, it stops and produces two new beams falling from the left and right.

    We need to count how many splitters are ever hit by a beam.

    The Simulation Approach

    Since beams always move downward, we don't need a complex queue or graph traversal. We can process the grid row by row.

    Imagine scanning the grid from top to bottom. We maintain a set of active columnsâ€”the columns where a beam is currently falling.

    1. Start: Initialize active_columns with the column of S.
    2. Scan: For each row r:
    3. Identify which columns in the current row have beams.
    4. If a beam hits ., it continues to row r+1 (same column).
    5. If a beam hits ^, it stops. We count it! Then, we add columns c-1 and c+1 to the set for row r+1.
    6. Repeat: Continue until the bottom.

    This approach is efficient because we only process each cell once.

    Key Insight: Merging! If two splitters send beams to the same spot (e.g., ^.^), the beams merge. By using a Set data structure for our active columns, this merging happens automatically.

    Part 2: Into the Multiverse ðŸŒŒ

    The manifold turns out to be quantum! Instead of one beam splitting, the timeline splits. We need to find the total number of timelines (paths) a single particle could take to reach the bottom.

    From Simulation to Dynamic Programming

    In Part 1, we only cared if a beam existed at (r, c). In Part 2, we care how many ways a particle can arrive at (r, c).

    This transforms the problem into path counting, similar to calculating coefficients in Pascal's Triangle or a game of Plinko.

    We use Dynamic Programming (DP). Let DP[c] be the number of timelines where the particle is at column c in the current row.

    1. Initialize: DP array of zeros, with DP[start_col] = 1.
    2. Propagate: For each row, calculate next_DP:
    3. If grid[r][c] is .: The count flows down. next_DP[c] += DP[c].
    4. If grid[r][c] is ^: The universe splits!
      • next_DP[c-1] += DP[c] (Left timeline)
      • next_DP[c+1] += DP[c] (Right timeline)
    5. Result: Sum all values in DP after the last row.

    Visualization: The Cascade

    Imagine the numbers flowing down like water:

    Row 0:       1
            / \
Row 1:     1   1    (Split!)
          / \ / \
Row 2:   1   2   1  (Merge!)

    At ^, the value N becomes N to the left and N to the right. At . , the value N stays N. When paths converge, we add the numbers.

    This explains why the answer gets so huge (27,055,852,018,812). The number of paths grows exponentially with the number of splitters!

    Why is this cool?

    This is a classic example of how a slight change in rules (counting beams vs counting paths) shifts a problem from simple set tracking to counting combinations. The DP approach solves it in O(Rows Ã— Cols) time, which is instantaneous even for trillions of paths!

    def solve():
    with open('input.txt', 'r') as f:
        lines = [line.strip() for line in f if line.strip()]

    height = len(lines)
    width = len(lines[0])

    # dp[c] stores the number of paths reaching column c at the current row
    # We can use a dictionary or list. List is faster.
    dp = [0] * width

    # Find start
    start_found = False
    for r in range(height):
        for c in range(width):
            if lines[r][c] == 'S':
                dp[c] = 1
                start_found = True
                break
        if start_found:
            break

    # Process row by row
    # We iterate from the start row to the end
    start_row = 0
    for r in range(height):
        if 'S' in lines[r]:
            start_row = r
            break

    for r in range(start_row, height):
        next_dp = [0] * width

        # Optimization: if all dp are 0, we can stop (though unlikely)
        if not any(dp):
            break

        for c in range(width):
            if dp[c] == 0:
                continue

            char = lines[r][c]
            count = dp[c]

            if char == '.' or char == 'S':
                # Beam continues straight down
                # Effectively to next_dp[c]
                next_dp[c] += count
            elif char == '^':
                # Beam splits
                left = c - 1
                right = c + 1

                if left >= 0:
                    next_dp[left] += count
                # else: side exit

                if right < width:
                    next_dp[right] += count
                # else: side exit

        dp = next_dp

    total_timelines = sum(dp)
    print(f"Part 2 Solution: {total_timelines}")

solve()
