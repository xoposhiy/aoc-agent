    Advent of Code 2025 â€” Day 7: Laboratories

    Summary

    • Problem: Simulate tachyon beams traveling downward through a grid. Beams travel down through '.' and 'S'. When a beam reaches a splitter '^', that beam stops and (for classical Part 1) emits two beams from the immediate left and right of the splitter that continue downward.
    • Part 1: Count the total number of beam splits that occur as a single steady stream of tachyons traverses the manifold.
    • Part 2: Quantum interpretation â€” a single particle takes both left and right choices at each splitter, creating multiple timelines. Count the number of distinct timelines after the particle finishes traversing the manifold.

    Key ideas and algorithmic approach

    Representation

    • I read the grid from input.txt and padded rows to equal width for simplicity.
    • Found the source 'S' coordinates (sx, sy).

    Part 1 (classical beams)

    • Model beams per-row as a set of active column positions.
    • Start with the column of 'S' at the row immediately below S.
    • For each row, process beams that arrive at that row.
    • If a beam hits a splitter '^', that beam stops (it does not continue down), and two new beams are emitted from the immediate left and right columns at the same row.
    • Because these new beams might immediately hit other splitters in the same row, we process the row until no further splitting can occur in that row (a fixed-point propagation), merging beams that land at the same cell.
    • Positions that are not splitters continue down to the next row.
    • Count splits whenever a beam hits a splitter.
    • This simulation is efficient because each row processes at most O(w) positions, and the grid has h rows, so overall O(h*w) operations. For the given input sizes (the Advent of Code grids), this is more than fast enough.

    Part 2 (quantum timelines)

    • Instead of sets of beam positions, we keep counts: how many timelines are currently at each column for a given row.
    • When k timelines hit a splitter at (x,y), they do not continue downward from x; rather they generate k timelines at (x-1,y) and k at (x+1,y). Again, these newly generated timelines may immediately encounter further splitters in the same row; we iterate within the row until no splitters receive timelines.
    • After stabilization, timelines at non-splitter columns move down to the next row with their counts preserved (and potentially merged if multiple paths lead to the same column).
    • After processing all rows, the sum of counts at the last row (or wherever timelines die out) is the number of distinct timelines.

    Why this works and efficiency

    • The key subtlety is that splitters are local and their emitted beams start at the same row. That means we must propagate splits inside a row until no new beams are on splitters â€” this can cause chain reactions across several adjacent splitters.
    • Using sets (Part 1) or integer counts (Part 2) and performing an intra-row stabilization loop ensures correctness and merges multiple beams/timelines that coincide. This avoids exponential blow-up in simulation while correctly counting merges.
    • Complexity is O(hw) per pass, with the inner row stabilization also bounded by O(w) per row, so still O(hw). The numbers involved in Part 2 can be large (they can grow exponentially in degenerate layouts), so we use Python integers to hold counts.

    Gotchas

    • Emitted beams start at the splitter's immediate left and right at the same row, not the next row.
    • When multiple beams/timelines reach the same cell, they merge â€” for Part 1 we deduplicate positions, for Part 2 we add counts.
    • Chains of splitters in the same row must be stabilized: a splitter can spawn beams that immediately hit another splitter in the same row.

    # Advent of Code 2025 Day 7 - Laboratories (Part 1)
# Read input from ./input.txt and simulate tachyon beams

from collections import deque

with open('input.txt') as f:
    grid = [list(line.rstrip('\n')) for line in f]

h = len(grid)
w = max(len(row) for row in grid)
# pad rows
for row in grid:
    if len(row) < w:
        row += ['.'] * (w - len(row))

# find S
sx = sy = None
for y in range(h):
    for x in range(w):
        if grid[y][x] == 'S':
            sx, sy = x, y
            break
    if sx is not None:
        break

if sx is None:
    print('No S found')
else:
    # We'll simulate beams as list of active beam positions (x,y)
    # Beams always move down. When a beam reaches splitter '^' at (x,y),
    # that beam stops and emits new beams from (x-1,y) and (x+1,y) continuing downward.
    # Careful: multiple beams can occupy same cell and should be merged.

    # Use a set for current beam x positions at a given row y.
    splits = 0
    # Start at row below S
    cur = set([sx])
    for y in range(sy+1, h):
        if not cur:
            break
        # For each x in cur, check cell at (x,y)
        next_beams = set()
        # If any beam hits splitter, it stops and creates beams from left and right at same row
        # But the emitted beams start from immediate left/right of the splitter and continue downward,
        # meaning their first occupied cell is at the same row y (not y+1)
        # So they will be included into next_beams for the same y iteration check.
        # To handle this, we process until fixed point within the row.
        active = set(cur)
        visited_in_row = set()
        while active:
            new_active = set()
            for x in active:
                if x < 0 or x >= w:
                    continue
                if (x,y) in visited_in_row:
                    continue
                visited_in_row.add((x,y))
                cell = grid[y][x]
                if cell == '^':
                    # split: spawn from x-1 and x+1 at same row
                    splits += 1
                    for nx in (x-1, x+1):
                        if 0 <= nx < w:
                            # emitted beam will continue down from nx at this same row: so we need to process (nx,y)
                            if (nx,y) not in visited_in_row:
                                new_active.add(nx)
                else:
                    # beam passes through empty or S; it continues down to next row from x
                    next_beams.add(x)
            active = new_active
        # set cur to next_beams for next row
        cur = next_beams
    print(splits)

    # Advent of Code 2025 Day 7 - Part 2
# We need to count number of distinct timelines (paths) of a single quantum particle that takes both left and right at splitters.
# Equivalent: Starting from S, particle moves down. At each splitter, the timeline splits: one timeline continues from left, another from right.
# Paths can merge (different sequences of choices can reach same position at same row). We need number of distinct final timelines (i.e., distinct endpoints after particle finishes).
# More formally, we simulate possible beam positions row by row, but instead of counting beams, we track how many ways (timelines) reach each (x,y).

from collections import defaultdict

with open('input.txt') as f:
    grid = [list(line.rstrip('\n')) for line in f]

h = len(grid)
w = max(len(row) for row in grid)
for row in grid:
    if len(row) < w:
        row += ['.'] * (w - len(row))

# find S
sx = sy = None
for y in range(h):
    for x in range(w):
        if grid[y][x] == 'S':
            sx, sy = x, y
            break
    if sx is not None:
        break

if sx is None:
    print('No S found')
else:
    # dp map: for current row y, counts[x] = number of timelines that are at column x and will move down into row y
    counts = defaultdict(int)
    counts[sx] = 1
    for y in range(sy+1, h):
        if not counts:
            break
        # process spread within row due to splitters emitting beams left/right at same row
        # We'll process events at positions (x) in this row; when a splitter receives k timelines, it stops them and emits k to left and k to right at same row.
        # We must allow repeated propagation from new positions if they hit more splitters. Use queue within the row.
        row_counts = defaultdict(int)
        # initially, incoming counts at this row are from counts
        for x,c in counts.items():
            row_counts[x] += c
        changed = True
        while True:
            new_row_counts = defaultdict(int)
            any_split = False
            for x,c in list(row_counts.items()):
                if c == 0:
                    continue
                if x < 0 or x >= w:
                    continue
                if grid[y][x] == '^':
                    # all c timelines split, they do NOT continue downward from this x; instead, they emit to x-1 and x+1 at same row
                    any_split = True
                    if x-1 >= 0:
                        new_row_counts[x-1] += c
                    if x+1 < w:
                        new_row_counts[x+1] += c
                    row_counts[x] = 0
            if not any_split:
                break
            # add new_row_counts into row_counts and loop
            for x,c in new_row_counts.items():
                row_counts[x] += c
        # after stabilization within row, remaining counts at positions (non-splitter positions) move down to next row
        counts = defaultdict(int)
        for x,c in row_counts.items():
            if c == 0: continue
            if x < 0 or x >= w: continue
            if grid[y][x] == '^':
                # shouldn't happen
                continue
            counts[x] += c
    # After processing all rows, the particle timelines have ended at last row positions in counts.
    # The question: number of different timelines active after a single particle completes all journeys.
    # That's the sum of counts across final positions.
    total = sum(counts.values())
    print(total)
