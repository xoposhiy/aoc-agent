    Advent of Code 2025, Day 7: Laboratories - Tachyon Manifold

    üî¨ Problem Summary

    You're trapped in a teleporter lab with a malfunctioning quantum tachyon manifold! The puzzle involves simulating tachyon beams through a grid-based manifold where:

    • A tachyon beam starts at position S and moves downward
    • When a beam hits a splitter (^), it splits into two beams going left and right
    • Beams pass through empty space (.) unchanged
    .......S.......     A beam enters at S and travels down
.......|.......     
.......^.......     When it hits a splitter (^)...
......‚Üô.‚Üò......     ...it splits into LEFT and RIGHT beams!

    üìä Part 1: Counting Beam Splits

    Task: Count how many times tachyon beams are split.

    Approach

    Track which columns contain active beams as we scan row by row:

    1. Start with a single beam at the starting column
    2. For each row, check what each beam encounters
    3. If it hits a splitter (^), increment split count and create two beams (left/right)
    4. If empty space, beam continues downward
    .......S.......
.......|.......
......|^|......    ‚Üê Split 1: creates 2 beams
......|.|......
.....|^|^|.....    ‚Üê Splits 2-3 (but middle column merges!)

    Key Insight

    Multiple beams can arrive at the same column position and effectively merge. We use a set to track unique beam positions.

    Answer: 1,628 splits ‚úÖ

    üåå Part 2: Quantum Timeline Splitting

    Plot Twist: It's actually a quantum tachyon manifold! A single particle takes BOTH paths at each splitter, creating parallel timelines.

    Task: Count the total number of timelines after the particle completes all journeys.

    The Many-Worlds Interpretation

             Timeline 1              Timeline 2
            ‚Üê                       ‚Üí
         (left)                   (right)

Each split DOUBLES the number of timelines!

    Algorithm Evolution

    Naive approach (DON'T DO THIS):

    • Simulate each individual timeline
    • With exponential growth, this would require ~27 trillion simulations!

    Smart approach:

    • Track how many timelines have a particle at each column position
    • When hitting a splitter, each of those N timelines branches into 2
    # Instead of tracking individual paths:
timelines = {column: count_of_timelines_at_column}

# When a particle at column c with N timelines hits a splitter:
left_count += N   # N timelines go left
right_count += N  # N timelines go right (total: 2N new timelines)

    Timeline Growth Visualization

    Row  2:    2 timelines  ##
Row  4:    4 timelines  ####
Row  6:    8 timelines  ########
Row  8:   13 timelines  #############     ‚Üê Not exactly 2^n due to merging!
Row 10:   20 timelines  ####################
Row 12:   26 timelines  ##########################
Row 14:   40 timelines  ########################################

    Answer: 27,055,852,018,812 timelines (27 trillion!) ‚úÖ

    üí° Key Insights

    1. The Merge Phenomenon

    The answer isn't simply 2^(number_of_splits) because timelines can merge! When two different paths lead to the same grid position and hit the same splitter, their combined timelines share that splitter.

    2. State Compression

    This is a classic counting paths problem. Instead of tracking billions of individual paths, we compress the state by tracking counts per position.

    3. Time & Space Complexity

    • Part 1: O(rows √ó cols) time, O(cols) space
    • Part 2: Identical! The "quantum" version runs just as fast because we're counting, not enumerating

    def solve_both_parts(filename='input.txt'):
    with open(filename) as f:
        grid = [list(line.rstrip()) for line in f]

    rows, cols = len(grid), len(grid[0])

    # Find start position 'S'
    start_col = next(c for r in range(rows) for c in range(cols) 
                     if grid[r][c] == 'S')

    # Part 1: Count splits (track positions as set)
    # Part 2: Count timelines (track position -> count dict)
    beams = {start_col}          # Part 1
    timelines = {start_col: 1}   # Part 2
    splits = 0

    for row in range(1, rows):
        new_beams, new_timelines = set(), {}

        for col in beams:
            if grid[row][col] == '^':
                splits += 1
                for dc in [-1, 1]:
                    if 0 <= col + dc < cols:
                        new_beams.add(col + dc)
            else:
                new_beams.add(col)

        for col, count in timelines.items():
            if grid[row][col] == '^':
                for dc in [-1, 1]:
                    if 0 <= col + dc < cols:
                        new_timelines[col + dc] = new_timelines.get(col + dc, 0) + count
            else:
                new_timelines[col] = new_timelines.get(col, 0) + count

        beams, timelines = new_beams, new_timelines

    return splits, sum(timelines.values())

    This puzzle beautifully demonstrates how the same underlying structure can yield vastly different numbers depending on whether you're counting events (splits) or possibilities (timelines)!

    def solve():
    with open('input.txt', 'r') as f:
        grid = [list(line.rstrip('\n')) for line in f]

    rows = len(grid)
    cols = len(grid[0]) if rows > 0 else 0

    start_col = None
    for col in range(cols):
        if grid[0][col] == 'S':
            start_col = col
            break

    if start_col is None:
        # Search all rows for S
        for row in range(rows):
            for col in range(cols):
                if grid[row][col] == 'S':
                    start_col = col
                    start_row = row
                    break
            if start_col is not None:
                break
    else:
        start_row = 0

    print(f"Grid: {rows} rows x {cols} cols")
    print(f"Start position: row={start_row}, col={start_col}")

    # Track active beams as (row, col) - all beams move downward
    # Use a set to track positions where tachyon beams exist
    # When a beam hits a splitter, it creates two new beams to left and right

    # We can track this row by row:
    # For each row, we know which columns have beams
    # When a beam reaches a splitter, remove the beam and add beams at col-1 and col+1

    # Track the columns that currently have active beams going down
    active_beams = {start_col}  # Set of column positions with active beams

    total_splits = 0

    # Start from the row after the start position and go downward
    for row in range(start_row + 1, rows):
        # Check what each active beam encounters at this row
        new_beams = set()

        for col in active_beams:
            if 0 <= col < cols:
                cell = grid[row][col]
                if cell == '^':
                    # Splitter: beam stops, two new beams emerge left and right
                    total_splits += 1
                    # Add beams to left and right
                    if col - 1 >= 0:
                        new_beams.add(col - 1)
                    if col + 1 < cols:
                        new_beams.add(col + 1)
                else:
                    # Empty space or other: beam continues
                    new_beams.add(col)

        active_beams = new_beams

        if not active_beams:
            break

    print(f"Total splits: {total_splits}")
    return total_splits

    def solve():
    with open('input.txt', 'r') as f:
        grid = [list(line.rstrip('\n')) for line in f]

    rows = len(grid)
    cols = len(grid[0]) if rows > 0 else 0

    # Find starting position S
    start_col = None
    start_row = 0
    for row in range(rows):
        for col in range(cols):
            if grid[row][col] == 'S':
                start_col = col
                start_row = row
                break
        if start_col is not None:
            break

    print(f"Grid: {rows} rows x {cols} cols")
    print(f"Start position: row={start_row}, col={start_col}")

    # Part 2: Count timelines
    # Each time a particle hits a splitter, the timeline splits into 2
    # But the particle can only be at one position in each timeline

    # Track: for each column position, how many timelines have a particle there
    # Start with 1 timeline at the starting position
    # When a particle at column c with N timelines hits a splitter:
    #   - N timelines go left (col c-1)
    #   - N timelines go right (col c+1)

    # Use a dictionary: col -> number of timelines with a particle at that column
    timelines = {start_col: 1}

    for row in range(start_row + 1, rows):
        new_timelines = {}

        for col, count in timelines.items():
            if 0 <= col < cols:
                cell = grid[row][col]
                if cell == '^':
                    # Splitter: particle goes both left and right in different timelines
                    # Each of the 'count' timelines splits into 2
                    left_col = col - 1
                    right_col = col + 1

                    if left_col >= 0:
                        new_timelines[left_col] = new_timelines.get(left_col, 0) + count
                    if right_col < cols:
                        new_timelines[right_col] = new_timelines.get(right_col, 0) + count
                else:
                    # Empty space: particle continues down in same column
                    new_timelines[col] = new_timelines.get(col, 0) + count

        timelines = new_timelines

        if not timelines:
            break

    # Total timelines is the sum of all timeline counts across all ending positions
    total_timelines = sum(timelines.values())

    print(f"Total timelines: {total_timelines}")
    print(f"Number of different ending columns: {len(timelines)}")

    return total_timelines
